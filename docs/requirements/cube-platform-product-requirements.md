# 产品需求文档 (PRD)

> **项目名称**：魔方学习与练习平台
> **版本**：v1.0
> **日期**：2025-02-16

## 一、项目背景与目标

1. 背景
   - 魔方爱好者需求：整合学习、练习与进阶功能的全栈平台，支持多平台访问（PC/移动端），逐步实现核心功能并扩展至智能硬件。

2. 目标
   - 打造一个涵盖魔方教程学习、用户进度管理、智能硬件联动的全栈应用。
   - 通过分阶段开发，确保每个阶段功能独立可用，降低开发风险。

## 二、用户需求分析

| **用户类型** | **核心需求**                                             |
| ------------ | -------------------------------------------------------- |
| **初学者**   | 系统化的入门教程、直观的 3D 公式演示、步骤分解学习         |
| **中级玩家** | 多种解法（CFOP、桥式）对比、公式标注和分类、练习进度追踪 |
| **高级玩家** | 智能魔方联机对战、自定义公式上传、多端同步数据           |

## 三、功能需求

### 阶段 1：基础功能（MVP）

#### 3.1.1 核心功能

1. 教程展示
   - 支持**层先法**、**CFOP**、**桥式**等教程分类，按步骤展示公式和文字说明。
   - 每个公式配套交互式 3D 魔方模型（支持旋转、步骤回放）。

2. 3D 模型交互
   - 使用 **Three.js** 渲染魔方，支持用户手动拖拽旋转或自动播放公式步骤。

#### 3.1.2 技术实现

- **前端**：**Vue3** + **Vue Router** + **Three.js**。
- **3D 模型数据**：JSON 格式存储公式步骤（例如：`{steps: ["U", "R", "F'"]}`）。

### 阶段 2：用户增强

#### 3.2.1 核心功能

1. 用户系统
   - 支持注册/登录（**JWT** 认证）、个人中心（公式标注、分类、自定义收藏夹）。

2. 公式管理
   - 用户可标记公式的难易程度、掌握进度，支持上传自定义解法。

#### 3.2.2 技术实现

- **后端**：**Node.js** + **Express** + **MongoDB**（存储用户数据、公式元数据）。

- 数据库设计：
  ```json
  // 用户表
  {
    "_id": ObjectId,
    "username": "user1",
    "password": "hashed_password",
    "formulas": [
      { "formula_id": "F001", "tags": ["掌握", "困难"], "custom_steps": [] }
    ]
  }
  ```

### 阶段 3：智能魔方（需评估可行性）

#### 3.3.1 核心功能

1. 硬件对接
   - 通过蓝牙连接 **Gan** / 魔域魔方，同步转动数据到平台。

2. 联机对战
   - 实时对战功能（**WebSocket** 实现），记录对战成绩。

#### 3.3.2 风险与建议

- 智能魔方厂商的 **API** 文档和 **SDK** 支持是关键，需优先调研（如 **Gan** 的开放接口）。
- 联机对战建议使用成熟框架（如 **Socket.io**），简化实时通信开发。

### 阶段 4：多端扩展

#### 3.4.1 核心功能

1. 微信小程序
   - 复用核心逻辑，适配小程序组件（如 **Taro** 框架）。

2. 移动端 App
   - **React Native** 开发，共享 80% 业务代码。

## 四、非功能需求

### 4.1 性能

- 3D 模型加载时间 ≤ 1s（使用 **WebGL** 优化和 **CDN** 加速静态资源）。
- 支持高并发用户访问，保证系统稳定性。

### 4.2 安全

- 用户密码加密存储（**bcrypt**）、**JWT** 有效期 ≤ 24h。
- 使用 **HTTPS** 保障数据传输安全。

### 4.3 兼容性

- PC 端支持 **Chrome** / **Firefox** / **Safari**，H5 适配 **iOS**/**Android** 主流机型。

## 五、技术选型

| **模块** | **技术方案**                                        |
| -------- | --------------------------------------------------- |
| **前端** | **Vue3** + **TypeScript** + **Pinia**（替代Vuex） + **Three.js**    |
| **后端** | **Node.js** + **Express** + **MongoDB**（Mongoose ODM）         |
| **部署** | **Docker** 容器化 + **Nginx** 反向代理 + **GitHub Actions CI/CD** |
| **测试** | **Jest**（单元测试）+ **Cypress**（E2E测试）                |

### 开发进度文档

#### 阶段 1：基础功能开发（预计4周）

| **周** | **任务**                 | **交付物**                             | **验收标准**                        |
| ------ | ------------------------ | -------------------------------------- | ----------------------------------- |
| 1      | 项目初始化 + 3D 魔方渲染  | **Vue3** 项目模板、静态 3D 模型演示页面       | 基础魔方模型可手动旋转/自动播放公式 |
| 2      | 教程页面开发（层先法）   | 完整的层先法教程页面（含步骤说明）     | 支持步骤切换、3D 模型同步更新        |
| 3      | 多解法支持（CFOP、桥式） | **CFOP** 和桥式教程页面                     | 不同解法可切换，3D 模型无冲突        |
| 4      | 性能优化 + 文档生成      | 文档站点（**GitHub Pages**）、性能测试报告 | 首屏加载 ≤ 1s，教程页面无卡顿         |

#### 阶段 2：用户系统开发（预计3周）

| **周** | **任务**                 | **交付物**                             | **验收标准**                   |
| ------ | ------------------------ | -------------------------------------- | ------------------------------ |
| 1      | 后端 **API** 开发（注册/登录） | 可运行的 **Node.js** **API**（Postman 测试通过） | 支持 **JWT** 签发、**MongoDB** 用户存储   |
| 2      | 前端用户界面（个人中心） | 用户仪表盘页面（公式收藏、标签管理）   | 数据与后端 **API** 实时同步          |
| 3      | 公式标注功能 + 测试      | 完整的用户系统（单元测试覆盖率 ≥ 70%）   | 用户可添加标签、上传自定义公式 |

#### 关键风险与应对建议

1. **3D 性能问题**
   - **应对**：使用 **Three.js** 的 `GLTFLoader` 加载轻量模型，启用 **WebGL** 压缩纹理。

2. **智能魔方接口缺失**
   - **应对**：阶段 3 开始时先模拟硬件数据（如虚拟魔方控制），降低依赖风险。

3. **多端兼容性**
   - **应对**：阶段 1 的 **H5** 页面严格使用响应式布局（**Flex** + **Grid**），提前测试移动端交互。
